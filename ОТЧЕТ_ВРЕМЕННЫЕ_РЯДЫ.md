## 1. Введение

### 1.1 Цель работы

Расширение существующего конвейера анализа сетевого трафика для реализации:
- Построения временного ряда из нормализованных/обогащённых сетевых событий
- Вычисления простого скользящего среднего (SMA) с настраиваемым окном
- Обнаружения аномалий относительно базовой линии SMA
- Генерации выходных файлов (CSV, JSON, PNG) и документации

### 1.2 Исходное состояние репозитория

Репозиторий уже содержал:
- Конвейер обработки PCAP → Zeek → нормализация → обогащение → ECS
- Выходной CSV-файл с 21 сетевым событием (`outputs/ecs/ecs_events.csv`)
- Модули статистического анализа (`baseline_modeling.py`, `timeseries_prep.py`)
- Маппинг в Elastic Common Schema

### 1.3 Используемая метрика

**Основная метрика:** количество соединений (connections) за временной интервал (bucket).

Эта метрика выбрана потому что:
- Охватывает все типы сетевых событий (conn + dns)
- Резкие всплески указывают на сканирование портов, DDoS или массовый обмен с C2
- Интуитивно понятна для аналитиков SOC

**Дополнительные метрики** (при наличии соответствующих полей):
- `nxdomain` — количество DNS-ответов NXDOMAIN (индикатор DGA)
- `failed` — количество неуспешных соединений (состояния S0, REJ, RSTO, RSTR и др.)

## 2. Теоретическая основа

### 2.1 Простое скользящее среднее (SMA)

SMA — это линейный фильтр, усредняющий последние W наблюдений:

```
SMA(t) = (1/W) * Σ x(t-i),  i = 0, ..., W-1
```

где:
- `x(t)` — значение метрики в момент t
- `W` — размер окна (window)

**Свойства SMA:**
- Сглаживает краткосрочные флуктуации
- Раскрывает тренды в данных
- Вычисляется за O(n) — пригоден для потоковой обработки

### 2.2 Правило обнаружения аномалий

Аномалия фиксируется при выполнении условия:

```
anomaly(t) = True,  если  x(t) > SMA(t) * K
```

где:
- `K` — множитель (multiplier), по умолчанию 2.0
- `SMA(t)` — скользящее среднее за последние W бакетов

**Интерпретация:** значение метрики превышает «типичный уровень» (SMA) более чем в K раз. Это указывает на резкое отклонение от недавнего нормального поведения.

### 2.3 Обработка граничных случаев

| Ситуация | Решение |
|----------|---------|
| Первые W-1 бакетов (SMA не полностью сформирован) | Используется `min_periods=1`, SMA вычисляется по доступным данным |
| SMA = NaN | Аномалия = False (консервативный подход) |
| Отсутствующие столбцы (нет dns_rcode, нет conn_state) | Лог предупреждения, метрика пропускается, скрипт продолжает работу |
| Один бакет | SMA = значение, x > x * K (при K >= 1) всегда False |

### 2.4 Почему SMA как базовая линия

| Причина | Пояснение |
|---------|-----------|
| **Прозрачность** | Аналитик может вручную проверить вычисления |
| **Нет допущений о распределении** | В отличие от z-score, не предполагает нормальность |
| **Адаптивность** | SMA «следует» за трендом — подстраивается к изменениям трафика |
| **Простота настройки** | Два параметра (W, K) с понятной семантикой |
| **Регуляторное соответствие** | Объяснимая логика: «значение превысило SMA в K раз» |

### 2.5 Нестационарность и сезонность — ограничения SMA

**Нестационарность:**
Временной ряд является стационарным, если его статистические свойства (среднее, дисперсия, автокорреляция) не меняются со временем. Реальный сетевой трафик **нестационарен**:
- Рабочие часы имеют более высокий трафик, чем ночь
- Развёртывание новых приложений меняет базовый уровень
- Атаки создают структурные сдвиги

SMA предполагает **локальную стационарность** внутри окна W. Если за период W происходит смена режима (например, начало рабочего дня), SMA может генерировать ложные срабатывания.

**Сезонность:**
Сезонность — предсказуемые повторяющиеся паттерны с фиксированным периодом:
- Ежечасно: DNS-запросы пиковые в начале часа (cron-задачи)
- Ежедневно: веб-трафик максимален в рабочие часы
- Еженедельно: понедельник утром — пик email-трафика

SMA **не моделирует сезонность**. Для учёта сезонных паттернов необходимы:
- Декомпозиция STL (Seasonal-Trend Decomposition using Loess)
- Модели SARIMA
- Сравнение с профилем «тот же час прошлой недели»

## 3. Реализация

### 3.1 Архитектура

```
┌──────────────┐    ┌──────────────┐    ┌─────────────┐    ┌──────────────┐
│    LOAD      │───▶│  AGGREGATE   │───▶│  SMA +      │───▶│    EXPORT    │
│  CSV events  │    │  Time bucket │    │  Anomaly    │    │ CSV/JSON/PNG │
└──────────────┘    └──────────────┘    └─────────────┘    └──────────────┘
```

### 3.2 Скрипт `scripts/time_series_sma_anomaly.py`

**Основные функции:**

| Функция | Назначение |
|---------|------------|
| `load_events()` | Загрузка CSV, парсинг timestamp как UTC datetime |
| `build_time_series()` | Агрегация событий в временные бакеты |
| `add_sma_and_anomalies()` | Вычисление SMA и пороговой детекции |
| `save_csv()` | Экспорт временного ряда в CSV |
| `save_anomaly_summary()` | Экспорт сводки аномалий в JSON |
| `save_plot()` | Построение графика matplotlib с маркерами аномалий |

**Аргументы командной строки:**

| Аргумент | Описание | По умолчанию |
|----------|----------|--------------|
| `--input` | Путь к входному CSV | Автоопределение |
| `--output-dir` | Директория для результатов | `outputs/` |
| `--bucket` | Размер временного бакета | `1min` |
| `--window` | Размер окна SMA (W) | `10` |
| `--multiplier` | Множитель аномалии (K) | `2.0` |

### 3.3 Автоопределение входных данных

Скрипт автоматически ищет CSV в следующем порядке:
1. `outputs/ecs/ecs_events.csv` (ECS формат, столбец `@timestamp`)
2. `outputs/final/enriched_network_events.csv` (нормализованный формат, столбец `timestamp`)

Поддерживаются оба формата — ECS и нормализованный.

### 3.4 Автоопределение столбцов

| Метрика | Столбцы ECS | Столбцы нормализованные |
|---------|------------|------------------------|
| Timestamp | `@timestamp` | `timestamp` |
| DNS rcode | `dns.response_code` | `dns_rcode` |
| Conn state | `zeek.connection.state` | `conn_state` |

| Время | Connections | SMA | Превышение |

- DNS-запрос `asdkjqwe.xyz` с ответом NXDOMAIN (индикатор DGA)

Это совпадает с активностью хоста **192.168.1.200**, который ранее был идентифицирован как подозрительный (DGA-паттерн запросов к случайным доменам).

### 4.4 Корреляция с другими метриками

В бакете 08:12:38 зафиксировано 3 неуспешных соединения (failed=3) — это три подряд попытки хоста 192.168.1.110 подключиться к IP-адресам из диапазона 44.230.90.0/24, все завершившиеся состоянием S0 (неполное TCP-рукопожатие). Хотя SMA-порог для connections не превышен (3 / 2.0 = 1.5), метрика `failed` подтверждает аномальность.

## 5. Ограничения и направления развития

### 5.1 Ограничения текущей реализации

| Ограничение | Описание | Возможное решение |
|-------------|----------|-------------------|
| **Малый объём данных** | 21 событие за 11 секунд — недостаточно для оценки производственной точности | Использование полноразмерных PCAP (CIC-IDS2017) |
| **Нестационарность** | SMA не учитывает долгосрочные тренды | EMA, разностное преобразование |
| **Сезонность** | Не моделируются дневные/недельные паттерны | STL-декомпозиция, SARIMA |
| **Равный вес наблюдений** | Все W точек в окне имеют одинаковый вес | EMA (экспоненциальное скользящее среднее) |
| **Одна метрика** | Аномалия определяется только по connections | Многомерная детекция (connections + nxdomain + failed) |
| **Статический порог** | K фиксирован для всех бакетов | Адаптивный порог на основе rolling std |

### 5.2 Направления развития

1. **Экспоненциальное скользящее среднее (EMA):** более чувствительно к недавним изменениям, снижает запаздывание.

2. **CUSUM (Cumulative Sum):** уже реализован в `scripts/cusum_detector.py` — детектирует постепенные сдвиги уровня.

3. **Многомерная аномальная детекция:** комбинирование connections, nxdomain и failed в единый показатель аномальности.

4. **Сезонная декомпозиция:** применение STL для выделения тренда, сезонной компоненты и остатка; аномалии выявляются по остатку.

5. **Автоматическая калибровка порога:** использование ROC-кривой или precision-recall для оптимального подбора K.

## Приложение А: Формулы

### SMA (Simple Moving Average)

```
SMA(t, W) = (1/W) * Σ_{i=0}^{W-1} x(t-i)
```

### Пороговая аномалия

```
anomaly(t) = {  True,   если x(t) > SMA(t, W) * K
             {  False,  иначе
```

### EMA (Exponential Moving Average) — для справки

```
EMA(t) = α * x(t) + (1 - α) * EMA(t-1),   α = 2 / (W + 1)
```
